# Breaking Change: koa-socket-2 Removal in 0.50.4

## Commit Reference
- Commit: `1e6df3a`
- Date: 2026-05-15
- Author: devill

## Motivation
`koa-socket-2` is unmaintained. Modernizing to SocketIO v4 native APIs enables:
- Current SocketIO features (adapters, namespaces)
- Simpler dependency tree
- Better compatibility with modern Node.js

## Architecture Change

### Before (0.50.2)
```
Browser SocketIO --> Koa App --> koa-socket-2 middleware --> SocketIO
                        |
                        --> User's Koa middleware (CORS, logging, etc.)
```

### After (0.50.4)
```
Browser SocketIO --> http.Server --> SocketIO (direct)
                         |
                         --- Koa app bypassed ---

Browser REST API --> Koa App --> User's middleware (still works)
```

## Current Separation of Concerns (Partial)

The codebase already has some separation, though `Server()` bundles everything together:

### Layer 1: Game Engine (`Master`) -- Transport Agnostic
- File: `src/master/master.ts`
- Handles move validation, state updates, game flow
- No dependency on SocketIO, Koa, or HTTP
- Used directly by AI/bots without transport layer

### Layer 2: Transport Interface (`TransportAPI`)
- File: `src/server/transport/socketio.ts` (exports `TransportAPI`)
- Pure interface: `{ send, sendAll }`
- Could be implemented by WebRTC, SSE, raw WebSocket, etc.

### Layer 3: Transport Implementation (`SocketIO`) -- Currently Only Option
- File: `src/server/transport/socketio.ts` (class `SocketIO`)
- Handles actual message delivery
- Currently hardcoded to HTTP + SocketIO

### Layer 4: `Server()` -- God Object
- File: `src/server/index.ts`
- Bundles Koa, HTTP server, SocketIO, REST API, Master
- Assumes single-process, single-server deployment

## Technical Details

### SocketIO CORS Mechanism Changed
- **Before:** Koa `@koa/cors` middleware handled CORS for all requests
- **After:** SocketIO's native `cors` option handles CORS for SocketIO traffic only
- **Impact:** `origins` parameter now uses `cors` npm package callback signature

### Koa Middleware Scope
| Middleware Type | Affected by Change? | Notes |
|----------------|-------------------|-------|
| `server.app.use(cors(...))` | Partial | Only affects REST, not SocketIO |
| `server.app.use(errorHandler)` | Partial | Only catches REST errors |
| Custom `/health` endpoint | No | REST endpoints unaffected |
| Security headers | Partial | Not sent on SocketIO responses |

## Code Migration

### Origins Function Signature
```typescript
// BEFORE (0.50.2) -- boolean return
origins: (origin: string | undefined) => boolean

// AFTER (0.50.4) -- callback style
origins: (origin: string | undefined, callback: (err: Error | null, allow: boolean) => void) => void

// ALTERNATIVE -- pass array
origins: ['http://localhost:5173']
```
